# Marketing AI

`/dashboard/marketing` (видимо в сайдбарі лише для `SUPPORT_ADMIN_EMAILS`,
той самий allowlist, що й `/dashboard/support-inbox`).

## Що робить зараз

Генерує чернетки трьох типів (пости в соцмережі, чернетки блог-статей,
брифи рекламних кампаній) через Claude, зберігає в `marketing_drafts`.
Власник редагує/копіює/схвалює вручну. **Нічого не публікується і жодні
гроші не витрачаються автоматично** — це свідоме рішення (обговорено з
власником): автопублікація й автономний рекламний бюджет без review —
надто високий ризик (репутаційний і фінансовий) для чогось, що працює без
людини на кожному кроці.

## Чого немає (свідомо, поки що)

- Реальної інтеграції з Google Ads / Meta Ads API — публікація й
  бюджетування залишаються ручними діями власника поза цим інструментом.
- Реальної інтеграції з Facebook/Instagram/LinkedIn API для авто-постингу.

## Що потрібно, щоб підключити реальну публікацію пізніше

Власник підтвердив (2026-07-19): наразі немає жодних бізнес-акаунтів
розробника чи API-доступу — починаємо з нуля. Коли буде готовність рухатись
далі:

1. **Meta (Facebook/Instagram)**: створити Meta Developer App +
   Business verification, отримати Page Access Token з правами
   `pages_manage_posts` (для постів) та Marketing API-доступ (для
   Ads) — це окремий, довший процес верифікації бізнесу від Meta.
2. **Google Ads**: Developer Token через Google Ads API Center + OAuth2
   credentials, прив'язка до реального рекламного акаунту з активним
   способом оплати.
3. Навіть після підключення API — залишити той самий чекпоінт: кнопка
   "Опубліковано" в `/dashboard/marketing` залишається ручною дією
   власника, а не автоматичним тригером. Максимум, що варто автоматизувати
   пізніше — сам виклик API одним кліком власника (замість
   копіювання/вставляння вручну), не рішення "коли і що публікувати".
